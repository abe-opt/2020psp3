## 課題  

以下の2つのアルゴリズムで文字列を検索するプログラムを作成する。  
- 力ずく法
- BM法

1. 使用するデータ  
以下のデータを使用する。ただし、検索対象文字列、検索する文字列は変更しても良い。  
    - StrOriginal: 検索対象データ
    - StrKey: 検索する文字列

2. 必須問題：実装する関数  
本課題では、以下の関数を実装する。C言語の標準ライブラリは使用しないこと。  
    (1) ForceSearch: 力ずく法で文字列を検索する。  
    [入力]  
    - char text[]: 検索対象文字列  
    - char key[]: 検索する文字列  

    [出力]  
    - return値：検索する文字列が出現した場所(ポインタ)。ただし、検索する文字列が見つからない場合はNULL。  

    (2) BMSearch: BM法で文字列を検索する。  
    [入力]  
    - char text[]: 検索対象文字列  
    - char key[]: 検索する文字列  
 
    [出力]  
    - return値：検索する文字列が出現した場所(ポインタ)。ただし、検索する文字列が見つからない場合はNULL。  

3. 補助関数  
なし

## ソースコードの説明
l15~16.探索対象,キー文字列の長さを調べる。
l18~31.比較開始位置はインデックス０から。一回の探索後スタート位置を１進める。探索対象をキーの文字列がはみ出すまで以下の探索を続ける。（ループ１）
l19~25.比較場所の文字を比較し、一致しなかったらループ２を抜ける。一致したらスタート位置のポインタを返す。（ループ２）
l27~30.キーのインデックスとキーの長さが一致したらループ１から抜け出す。

l41~42.探索対象,キー文字列の長さを調べる。
l45~47.ずらし量テーブルの作成。
l49~51.キーの文字のずらし量テーブルの作成。
l53~69.最初の探索対象の比較位置は、探索対象のキーの長さ-1からする。探索対象の長さを比較位置が超えたら探索終了。以下の探索を行う。
l54.キーの末尾から比較する。
l55~62.文字を比較していき、全て比較出来たら、探索対象のインデックスのポインタを返す。
l60~61.キーと探索対象のインデックス両方を１ずつ減らしていく。
l64~68.前回よりも比較位置が戻るならば、キーを一つ後ろにずらす。それ以外の時は、テーブルに従って比較位置をずらす。
